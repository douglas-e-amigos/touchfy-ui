### User Story US01 - Registrar novo usuário

| 📌 User Story - US01 | Registrar novo usuário |
|---------------------|--|
| **Título** | Registrar novo usuário |
| **Identificação** | US01 - Registrar novo usuário |
| **Story** | Como *guest*, quero *criar* uma nova conta de usuário, para *acessar as principais funcionalidades do sistema*, como ouvir músicas, criar playlists, etc. |
| **Requisitos Relacionados** | RF01 |
|**Regras de Negócio** | - Somente usuários convidados (não logados) podem cadastrar uma nova conta<br>- O campo de e-mail deve ser único e ser um e-mail válido<br>- O campo de nome de usuário deve único, deve iniciar com alfabéticos e ter tamanho máximo de 100 caracteres<br>-A senha deverá ser maior/igual que oito caracteres e deverá ter pelo menos uma letra maiúscula, uma letra minúscula, um número e um símbolo<br>- A data de nascimento deverá ser digitada no formato dd/mm/yyyy mas enviada ao servidor como yyyy-mm-dd<br>- A data de nascimento deverá ser anterior aos últimos 15 anos |

### User Story US02 - Login de usuário
| 📌 User Story - US02 | Login de usuário |
|---------------------|--|
| **Título** | Login de usuário |
| **Identificação** | US02 - Login de usuário |
| **Story** | Como *usuário*, quero *acessar minha conta com e-mail e senha*, para *utilizar as funcionalidades do sistema*. |
| **Requisitos Relacionados** | RF01 |
|**Regras de Negócio** | - O e-mail inserido deverá ser um e-mail válido<br>- A senha inserida deverá ser uma senha válida, com pelo menos um caracter de cada tipo: minúsculo, maiúsculo, número e símbolo<br>- A senha deverá ter pelo menos oito caracteres<br>- Somente usuários não logados e com conta cadastrada poderão realizar login |
| **Critérios de Aceitação** | - **Dado** que estou na tela de login, **quando** informo e-mail e senha válidos, **então** o sistema deve permitir o acesso e redirecionar para a tela principal.<br>- **Dado** que informo credenciais inválidas, **quando** tento autenticar, **então** o sistema deve exibir mensagem de erro e não permitir o acesso.<br> |
| **Testes de Aceitação** | <ul><li>TA02.01 - Login válido.<ol><li>Acessar a tela de login</li><li>Preencher e-mail válido</li><li>Preencher senha válida</li><li>Clicar em entrar</li><li>Verificar redirecionamento para a tela principal</li></ol></li><li>TA02.02 - E-mail não informado.<ol><li>Acessar a tela de login</li><li>Deixar o campo e-mail vazio</li><li>Preencher senha</li><li>Verificar que o sistema não permite o envio</li><li>Forçar envio</li><li>Verificar mensagem de erro</li></ol></li><li>TA02.03 - Senha não informada.<ol><li>Acessar a tela de login</li><li>Preencher e-mail</li><li>Deixar senha vazia</li><li>Verificar que o sistema não permite o envio</li><li>Forçar envio</li><li>Verificar mensagem de erro</li></ol></li><li>TA02.04 - Credenciais inválidas.<ol><li>Acessar a tela de login</li><li>Preencher e-mail ou senha inválidos</li><li>Clicar em entrar</li><li>Verificar mensagem de erro</li><li>Verificar que o acesso não foi permitido</li></ol></li></ul> |

### User Story US03 - Editar dados do usuário
| 📌 User Story - US03 | Editar dados do usuário |
|---------------------|--|
| **Título** | Editar dados do usuário |
| **Identificação** | US03 - Editar dados usuário |
| **Story** | Como *usuário autenticado*, quero *editar meus dados (nome de usuário, e-mail, senha e foto)*, para *manter minhas informações atualizadas*. |
| **Requisitos Relacionados** | RF01 |
|**Regras de Negócio** | - A imagem deverá estar no formato jpg/png/webp e ter no máximo 10MB, com dimensões máximas de 400x400px<br>- Somente o usuário dono da conta poderá alterar seu perfil |
| **Critérios de Aceitação** | - **Dado** que estou autenticado, **quando** acesso a tela de edição, **então** devo visualizar meus dados atuais.<br>- **Dado** que altero dados válidos, **quando** salvo, **então** o sistema deve persistir as alterações.<br>- **Dado** que seleciono uma imagem válida (jpg, png, webp) dentro do limite permitido, **quando** faço upload, **então** o sistema deve exibir preview.<br>- **Dado** que envio imagem inválida ou acima do limite, **quando** tento salvar, **então** o sistema deve exibir erro.<br> |
| **Testes de Aceitação** | <ul><li>TA03.01 - Edição válida.<ol><li>Acessar tela de edição</li><li>Alterar campos</li><li>Salvar</li><li>Verificar persistência</li></ol></li><li>TA03.02 - Upload de imagem válido.<ol><li>Acessar tela</li><li>Selecionar imagem válida (jpg/png/webp)</li><li>Verificar preview</li><li>Salvar</li><li>Verificar persistência</li></ol></li><li>TA03.03 - Upload de imagem inválido.<ol><li>Acessar tela</li><li>Selecionar arquivo inválido ou acima do limite</li><li>Tentar salvar</li><li>Verificar mensagem de erro</li></ol></li><li>TA03.04 - Campo obrigatório vazio.<ol><li>Acessar tela</li><li>Remover valor de campo obrigatório</li><li>Verificar bloqueio de envio</li><li>Forçar envio</li><li>Verificar mensagem de erro</li></ol></li></ul> |

### User Story US04 - Desativar conta
| 📌 User Story - US04 | Desativar conta |
|---------------------|--|
| **Título** | Desativar conta |
| **Identificação** | US04 - Desativar conta |
| **Story** | Como *usuário autenticado*, quero *desativar minha conta confirmando minha senha*, para *encerrar meu uso do sistema com segurança*. |
| **Requisitos Relacionados** | RF01 |
|**Regras de Negócio** | - Somente o usuário dono da conta ou um administrador do sistema poderão desativar a conta do dito usuário<br>- A desativação deverá ser informada por e-mail quando ocorrer |
| **Critérios de Aceitação** | - **Dado** que estou autenticado, **quando** solicito desativação da conta, **então** o sistema deve solicitar confirmação por senha.<br>- **Dado** que informo a senha correta, **quando** confirmo, **então** o sistema deve desativar a conta.<br>- **Dado** que informo senha incorreta, **quando** tento confirmar, **então** o sistema deve exibir erro e não desativar.<br>- **Dado** que não informo a senha, **quando** tento confirmar, **então** o sistema deve bloquear e exibir erro.<br>- **Dado** que o envio é forçado sem senha, **quando** chega ao servidor, **então** o sistema deve rejeitar a operação. |
| **Testes de Aceitação** | <ul><li>TA04.01 - Desativação válida.<ol><li>Acessar funcionalidade</li><li>Informar senha correta</li><li>Confirmar</li><li>Verificar que a conta foi desativada</li></ol></li><li>TA04.02 - Senha incorreta.<ol><li>Acessar funcionalidade</li><li>Informar senha incorreta</li><li>Confirmar</li><li>Verificar mensagem de erro</li><li>Verificar que a conta não foi desativada</li></ol></li><li>TA04.03 - Senha não informada.<ol><li>Acessar funcionalidade</li><li>Deixar senha vazia</li><li>Verificar bloqueio</li><li>Forçar envio</li><li>Verificar erro</li></ol></li></ul> |
