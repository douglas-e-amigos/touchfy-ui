### User Story - US26 | Criar uma nova playlist

| 📌 User Story - US26 | Criar uma nova playlist |
|---------------------|--|
| *Título* | Criar uma nova playlist |
| *Identificação* | US26 - Criar uma nova playlist |
| *Story* | Como usuário, quero criar uma nova playlist informando um nome, opcionalmente uma descrição e definindo sua visibilidade, para organizar minhas músicas. |
| *Requisitos Relacionados* | RF01 |
|**Regras de Negócio** | - Somente um usuário logado pode criar uma playlist<br>- O nome de uma playlist deve ser um texto válido e não vazio<br>- A descrição da playlist deve ser um texto que pode ser nulo<br> - A visiblidade da playlist deve ser uma opção entre: público, privado e protegido |
| *Critérios de Aceitação* | - **Dado** que estou autenticado, **quando** acesso a tela de criação de playlist, **então** devo conseguir informar o nome, uma descrição opcional e definir a visibilidade (pública ou privada).<br><br>- **Dado** que informo um nome válido, **quando** confirmo a criação, **então** a playlist deve ser criada com sucesso.<br><br>- **Dado** que não informo um nome, **quando** tento criar a playlist, **então** o sistema deve exibir uma mensagem de erro.<br><br>- **Dado** que informo uma descrição, **quando** a playlist é criada, **então** a descrição deve ser salva corretamente.<br><br>- **Dado** que seleciono uma visibilidade, **quando** a playlist é criada, **então** ela deve respeitar a configuração escolhida. |
| *Testes de Aceitação* | <ol><li><strong>TA01.01 - Criação de playlist com dados válidos.</strong><ol><li>Acessar a tela de criação de playlist</li><li>Preencher o campo nome com um valor válido</li><li>Preencher o campo descrição (opcional)</li><li>Selecionar visibilidade pública</li><li>Clicar em criar playlist</li><li>Verificar que o sistema cria a playlist com sucesso</li><li>Verificar que a playlist aparece na lista do usuário</li><li>Verificar que a visibilidade está definida como pública</li></ol></li><li><strong>TA01.02 - Criação de playlist sem descrição.</strong><ol><li>Acessar a tela de criação de playlist</li><li>Preencher o campo nome com um valor válido</li><li>Deixar o campo descrição vazio</li><li>Selecionar visibilidade privada</li><li>Clicar em criar playlist</li><li>Verificar que o sistema cria a playlist com sucesso</li><li>Verificar que a playlist aparece na lista do usuário</li><li>Verificar que a visibilidade está definida como privada</li></ol></li><li><strong>TA01.03 - Nome da playlist não informado.</strong><ol><li>Acessar a tela de criação de playlist</li><li>Deixar o campo nome vazio</li><li>Preencher os demais campos corretamente</li><li>Tentar criar a playlist</li><li>Verificar que o sistema não permite a criação</li><li>Forçar o envio (ex: via requisição manual)</li><li>Verificar que o sistema exibe mensagem de erro</li><li>Verificar que a playlist não foi criada</li></ol></li><li><strong>TA01.04 - Persistência da descrição.</strong><ol><li>Acessar a tela de criação de playlist</li><li>Preencher o campo nome com um valor válido</li><li>Preencher o campo descrição</li><li>Selecionar visibilidade pública</li><li>Criar a playlist</li><li>Acessar a playlist criada</li><li>Verificar que a descrição exibida é igual à informada</li></ol></li><li><strong>TA01.05 - Persistência da visibilidade.</strong><ol><li>Acessar a tela de criação de playlist</li><li>Preencher o campo nome com um valor válido</li><li>Selecionar visibilidade privada</li><li>Criar a playlist</li><li>Acessar a playlist criada</li><li>Verificar que a playlist está configurada como privada</li><li>Verificar que outros usuários não conseguem visualizar a playlist</li></ol></li></ol> |

### User Story US31 - Reprodução de playlist e controle de reprodução

| 📌 User Story - US31 | Reprodução de playlist e controle de reprodução |
|---------------------|--|
| *Título* | Reproduzir playlists e controle de reprodução |
| *Identificação* | US31 - Reprodução de playlist e controle de reprodução |
| *Story* | Como usuário, quero reproduzir playlists e controlar sua execução (pausar, avançar, voltar e definir a ordem), para ouvir uma sequência contínua de músicas conforme minha preferência. |
| *Requisitos Relacionados* | RF06 |
|**Regras de Negócio** | - Quando pública, qualquer usuário pode reproduzir a playlist<br>- Quando protegida, apenas os usuários selecionados podem reproduzir a playlist<br>- Quando privada, somente os usuários donos podem reproduzir a playlist |
| *Critérios de Aceitação* | - **Dado** que estou autenticado, **quando** seleciono uma playlist e aciono reproduzir, **então** o sistema deve iniciar pela primeira música e definir a playlist como fila ativa.<br><br>- **Dado** que uma playlist está em reprodução, **quando** uma música termina, **então** a próxima da playlist deve iniciar automaticamente.<br><br>- **Dado** que estou reproduzindo uma playlist, **quando** aciono pausar, **então** a execução deve parar mantendo posição e tempo atual.<br><br>- **Dado** que a reprodução está pausada, **quando** aciono reproduzir, **então** deve continuar do ponto exato na playlist.<br><br>- **Dado** que estou reproduzindo uma playlist, **quando** avanço a música, **então** o sistema deve ir para a próxima da fila.<br><br>- **Dado** que estou reproduzindo uma playlist, **quando** volto a música, **então** deve reiniciar a atual ou ir para a anterior conforme o tempo de reprodução.<br><br>- **Dado** que seleciono modo sequencial, **quando** a playlist é executada, **então** a ordem original deve ser respeitada até o fim.<br><br>- **Dado** que seleciono modo aleatório, **quando** a playlist é executada, **então** as músicas devem ser embaralhadas sem repetição até completar a lista.<br><br>- **Dado** que a playlist chega ao fim, **quando** não há mais músicas, **então** a reprodução deve ser encerrada.<br><br>- **Dado** que a playlist está vazia, **quando** tento reproduzir, **então** o sistema deve impedir a ação. |
| *Testes de Aceitação* | <ol><li><strong>TA06.01 - Reprodução de playlist.</strong><ol><li>Selecionar uma playlist</li><li>Clicar em reproduzir</li><li>Verificar início pela primeira música</li></ol></li><li><strong>TA06.02 - Execução contínua.</strong><ol><li>Reproduzir playlist</li><li>Aguardar fim da música</li><li>Verificar início automático da próxima</li></ol></li><li><strong>TA06.03 - Pausar e retomar.</strong><ol><li>Reproduzir playlist</li><li>Pausar</li><li>Retomar</li><li>Verificar continuidade do ponto anterior</li></ol></li><li><strong>TA06.04 - Avançar música.</strong><ol><li>Reproduzir playlist</li><li>Clicar em próxima</li><li>Verificar avanço na fila</li></ol></li><li><strong>TA06.05 - Voltar música.</strong><ol><li>Reproduzir playlist</li><li>Clicar em voltar</li><li>Verificar reinício ou música anterior</li></ol></li><li><strong>TA06.06 - Modo sequencial.</strong><ol><li>Ativar modo sequencial</li><li>Reproduzir playlist</li><li>Verificar ordem original</li></ol></li><li><strong>TA06.07 - Modo aleatório.</strong><ol><li>Ativar modo aleatório</li><li>Reproduzir playlist</li><li>Verificar ordem embaralhada sem repetição</li></ol></li><li><strong>TA06.08 - Fim da playlist.</strong><ol><li>Reproduzir até última música</li><li>Verificar parada da execução</li></ol></li><li><strong>TA06.09 - Playlist vazia.</strong><ol><li>Acessar playlist vazia</li><li>Tentar reproduzir</li><li>Verificar bloqueio da ação</li></ol></li></ol> |
